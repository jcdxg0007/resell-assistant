## 八、端到端流程测试（E2E）

### 流程 A：闲鱼选品 → 发布
```
1. 在闲鱼选品页面发现高分商品
2. 查看十维度评分和定价建议
3. 在闲鱼工作台创建发布草稿（标题改写 + 图片处理）
4. 发布到闲鱼
```
⚠️ 需要：已配置闲鱼账号 + Playwright 浏览器可用

### 流程 B：收到订单 → 履约
```
1. Celery 自动检测到新订单
2. 系统提示源平台下单
3. 物流信息同步到闲鱼
4. 钉钉推送通知
```
⚠️ 需要：真实订单 + 钉钉 webhook 已配置

### 流程 C：小红书选品 → 内容发布
```
1. 在小红书选品页面发现热门话题
2. AI 生成笔记草稿
3. 编辑优化后发布
```
⚠️ 需要：已配置小红书账号 + LLM API Key

## 九、需要提前配置的环境变量

以下功能需要对应的环境变量才能完整测试：

| 功能 | 所需环境变量 | 添加到 |
|------|-------------|--------|
| AI 内容生成/建议 | `LLM_API_KEY` | backend + celery |
| 钉钉推送 | `DINGTALK_WEBHOOK_URL`, `DINGTALK_SECRET` | backend + celery |
| 邮件推送 | `SMTP_HOST`, `SMTP_USER`, `SMTP_PASSWORD` | backend + celery |

## 十、测试优先级建议

```
第 1 轮（立即可测）：
  ✅ 基础设施验证 → 认证系统 → 各页面加载 → API 文档浏览

第 2 轮（配置 LLM Key 后）：
  ✅ AI 运营建议 → 小红书内容生成 → 客服 AI 预回复

  ✅ 选品数据采集 → 闲鱼发布 → 小红书发布

第 4 轮（真实运营）：
```
