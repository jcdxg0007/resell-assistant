# 账号入口隔离 SOP

> 每次往爬虫池加新账号时**必须遵守**的标准流程。
> 目的：防止新号一登录就被平台风控关联到既有的"低信任池"。

---

## 0. 为什么要这么做

平台风控（PDD / 淘宝 / 1688）建立这样的图谱：

```
device_fingerprint  ↔  [account_A, account_B, account_C, ...]
login_ip            ↔  [account_A, account_B, account_C, ...]
nano_fp / 设备 ID    ↔  [account_A, account_B, account_C, ...]
```

只要任意两个账号在**同一台设备 + 同一个 IP**登录过，它们在平台风控图谱里就被永久绑在同一个节点上。任何一个号触发深度审查，**整个节点上的号一并被打低信任标签**。

⚠️ 历史已发生的关联无法回滚（PDD 服务端日志保留期通常 ≥ 6 个月），只能阻止新增。

---

## 1. 三种登录入口的风险等级

| 入口类型 | 风险等级 | 说明 |
|---|---|---|
| **A：主力机 + 家庭 WiFi** | 🔴 高 | **永远不要**用这个登录新爬虫号。已存在的关联池**已经在这里**，再加号 = 继续污染池子 |
| **B：隔离机 + 4G 流量** | 🟢 低 | 推荐。每个登录环节都跟主力机不交叉 |
| **C：买现成已养号 cookies** | 🟢🟢 极低 | 最佳。登录指纹是卖家的，不进你的图谱 |

---

## 2. 入口 B：隔离机 + 4G 流量（推荐）

### 2.1 一次性准备（一台旧手机就够）

需要的设备：

- 一台不用的安卓手机或 iPhone（性能不重要，能正常装 App 即可）
- 一张独立 SIM 卡（30 元/月套餐就行，**不要**用你主力手机的卡）
- **不要**用任何家里的 WiFi 给它联网

不需要：

- 不需要 root / 越狱
- 不需要装代理软件
- 不需要任何特殊改造

### 2.2 每次加号的标准流程

```
[1] 隔离机开机
    └─ 关闭 WiFi
    └─ 打开 4G/5G 数据流量

[2] （可选但推荐）飞行模式开关一次刷新 IP

[3] 打开拼多多/淘宝/1688 App（隔离机上单独装的，不要用账号登录过的 App）

[4] 用新号的手机号 + 验证码登录

[5] 在 App 里**正常使用 5-10 分钟**：
    - 浏览 3-5 个商品
    - 加购物车 1-2 件
    - 完成实名认证（如果还没做）
    - 绑定支付方式
    （这步是建立账号画像，不能跳）

[6] 提取 cookies：
    - PDD：用电脑扫 PDD H5 二维码登录 H5 网页（仅一次，仅为了导出 cookies）
      或者：用 Frida hook PDD App 提取（高阶）
    - 1688/淘宝：H5 登录 → Cookie-Editor 导出
```

### 2.3 提取 cookies 的"必要交叉点"

第 6 步**确实**会让你电脑接触一次该账号——但只要：

- 只是**一次性**导出，不长期使用
- 导出完立刻关闭网页/退出登录
- 不在主力机上**真正登录使用**这个号

平台风控对"扫码登录 H5 一次"的权重远低于"持续在 X 设备使用"。这点折衷是可接受的。

更彻底的方案：**直接在隔离机上**用 USB 线连电脑跑 ADB，从隔离机的 PDD App SQLite 直接读 cookies。但这是高阶玩法，简单做法已经够用。

### 2.4 之后续期/补充 cookies

- **新号**首次入库后，每次平台 cookies 过期前在**隔离机的 App** 上访问一下（让 App 续期）
- 续期时**禁止**用主力机访问该账号
- 如果某天必须用主力机访问（紧急排错），那个号要从爬虫池下线 24 小时再回来（让平台风控的"短期异常"窗口过去）

## 3. 入口 C：买现成的已养号

最省事，适合不想自己养的场景。

### 3.1 关键采购要求

跟卖家明确：

- ✅ **必须**：卖家自己用他的设备登录、激活、养号 ≥ 7 天
- ✅ **必须**：提供完整 cookies（Cookie-Editor 导出 JSON 或 storage_state）
- ✅ **必须**：账号已实名 + 绑支付方式 + 有过下单记录
- ❌ **不要**："注册号" / "0 天号" / "可以你自己登录" 这种 = 等于让你污染指纹

### 3.2 价格参考

| 类型 | 价格 | 推荐度 |
|---|---|---|
| 注册即卖（白号） | 5-10 元 | ❌ 别买 |
| 养号 7-14 天 | 15-30 元 | 🟡 一般 |
| 养号 ≥ 30 天，有下单记录 | 30-80 元 | ✅ 推荐 |
| 高质量老号（半年+） | 100-200 元 | ✅✅ 高频爬虫主力推荐 |

### 3.3 导入流程

cookies 拿到后用现有脚本导入：

```bash
cd /home/devbox/project/backend
python3 scripts/pdd_import_cookies.py \
  --mobile <账号手机号> \
  --files /tmp/<from_seller>.json
```

之后立刻跑健康体检：

```bash
python3 scripts/test_pdd_account.py --suffix <last4>
```

返回 ≥ 5 商品就是健康号。

## 4. 已经污染的现有号怎么办

**接受现状，不要硬救**：

- 主力机 + 家庭 WiFi 上登录过的号**已经在关联池里**，无法解除关联
- 但只要这个池子里**有 1-2 个高信任号**（养号到位 / 有真实购物），整个池子的水位会被抬高
- 所以补救路径：**通过入口 C 买几个高质量号补进池子**，让池子整体看起来像"一群正常用户"，而不是"一群可疑用户"

## 5. 检查清单（每次加号前过一遍）

```
□ 这次登录用的是隔离机吗？      （主力机 = 立刻停止）
□ 用的是 4G 流量吗？             （家里 WiFi = 立刻停止）
□ 隔离机上的 App 是干净的吗？    （登过别的号 = 卸载重装）
□ 登录后在 App 里活动了 ≥ 5 分钟？（没有 = 当作白号继续养）
□ 已经实名 + 绑支付？            （没有 = 立刻去做）
□ 提取 cookies 时是否一次性、不留连接？
□ 录入数据库后立刻跑了健康体检？
```

---

## 6. 不要做的事

- ❌ **不要**用主力机注册任何新爬虫号
- ❌ **不要**在多个号之间在主力机上来回切换登录
- ❌ **不要**用同一台手机的 SIM 卡注册大量号（运营商也有反作弊）
- ❌ **不要**所有号都用同一个时间段集中注册/登录
- ❌ **不要**给隔离机连家庭 WiFi（即便"就一下下"）
- ❌ **不要**省略养号步骤直接入库——白号入库后一个月内基本只能跑出空数据
