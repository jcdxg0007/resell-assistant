# 转卖助手 - 开发文档 v3.0

> 多平台信息差电商工具，核心价值：AI驱动选品 + 多渠道销售自动化 + 智能运营

---

## 一、项目概述

### 1.1 核心定位

系统分为两大主体：

| 主体 | 定位 | 核心能力 |
|------|------|---------|
| **选品助手** | 发现什么值得卖 | 跨平台比价、信息差分析、内容选品、趋势发现 |
| **销售助手** | 把东西卖出去 | 多平台发布、订单履约、物流同步、账号管理 |

### 1.2 多平台策略

不同平台的电商逻辑完全不同，系统需要分别适配：

| 平台 | 类型 | 起号逻辑 | 开发重点 |
|------|------|---------|---------|
| **闲鱼** | 货架电商 | 搜索排名驱动，标题SEO+擦亮+低价策略 | 比价引擎、批量发布、价格监控 |
| **小红书** | 内容电商 | 内容种草驱动，笔记质量+视觉审美+话题热度 | 爆文分析、内容生成、笔记发布 |
| **抖音** | 内容电商 | 短视频/直播驱动，商品卡+视频素材+算法推荐 | 商品卡管理（后期扩展） |

### 1.3 运营模式

- **渐进式自动化**：前期半自动（人工确认），后期AI自动执行+异常时人工介入
- **零库存**：代发模式，不囤货；虚拟商品自动发货
- **信息差+内容差**：闲鱼靠信息差定价，小红书靠内容溢价
- **AI驱动运营**：每日自检、运营日报、行动建议、客服预回复

### 1.4 利润目标

月利润 ¥100,000+，通过多平台分摊实现：

| 渠道 | 账号数 | 日均单量 | 月利润占比 |
|------|--------|---------|-----------|
| 闲鱼 | 3~4个 | 10~15单 | 40% |
| 小红书 | 3~4个 | 8~12单 | 35% |
| 抖音 | 1~2个 | 3~5单 | 15% |
| 其他(淘宝企业店等) | 1个 | 3~5单 | 10% |

### 技术选型

| 层级 | 技术 | 理由 |
|------|------|------|
| 前端 | React 18 + Ant Design 5 + Vite | 企业级组件库，适合多模块管理后台，响应式适配移动端 |
| 后端 | Python 3.12 + FastAPI | 异步高性能，爬虫/ML/内容处理生态丰富 |
| 数据库 | PostgreSQL | 商品/订单/内容关系型数据 |
| 缓存 | Redis | 任务队列、价格缓存、Session |
| 任务队列 | Celery + Redis Broker | 定时爬取、内容分析、AI自检、异步任务 |
| 浏览器自动化 | Playwright + stealth | 数据采集、辅助发布（闲鱼+小红书）、客服消息采集 |
| 图像识别 | CLIP (open_clip) | 同款匹配、内容视觉分析 |
| 图片处理 | Pillow + 自定义管线 | 图片差异化、封面图生成、评论区实拍图加工 |
| AI能力 | LLM API (GPT/Claude等) | 运营建议生成、客服预回复、内容改写、笔记草稿 |
| 物流查询 | 快递100 / 快递鸟 API | 物流追踪 |
| 消息推送 | 钉钉Webhook + 邮件 | 双通道保障 |
| 部署 | Docker + Docker Compose | 轻量部署，支持远程管理 |

### 架构设计原则（面向规模化）

- **平台 = 插件**：每个销售平台是独立模块，后续加抖音/淘宝只需开发新插件
- **多人协作支持**：系统支持多用户登录+权限管理，可分配账号给助手
- **可配置自动化**：每个操作有"人工确认/AI自动"开关，逐步放权
- **账号无限扩展**：添加新账号=填一个表单，系统自动接管

## 三、选品助手

选品助手有两个引擎，分别服务货架电商和内容电商：

### 3.1 跨平台比价引擎（服务闲鱼）

#### 数据获取策略

| 数据源 | 主力方式 | 补充方式 |
|--------|---------|---------|
| 拼多多 | 多多进宝 API | Playwright |
| 淘宝 | 淘宝联盟 API | Playwright |
| 闲鱼 | Playwright | — |

#### 商品发现方式

- **自动发现**：按预设品类关键词定时搜索
- **即时查询**：手动输入关键词或商品链接实时查询

#### 同款商品识别流水线

```
Step 1: 关键词匹配 + 标题相似度 + 价格区间 → 缩小到 20~50 个
Step 2: 感知哈希 (pHash) → 缩小到 5~15 个
Step 3: CLIP 向量相似度 → 精确判定 (>0.85同款 / 0.70~0.85人工复核)
Step 4: 人工复核（可选）
```

#### 十维度评分体系

| # | 维度 | 权重 | 核心问题 | 数据来源 |
|---|------|------|---------|---------|
| 1 | 闲鱼在售链接数 | 15% | 竞争有多激烈？ | 闲鱼 |
| 2 | 价格离散度 (CV) | 15% | 买家价格预期清不清晰？ | 闲鱼 |
| 3 | 闲鱼"想要"热度 | 12% | 真的有人想买吗？ | 闲鱼 |
| 4 | 源平台销量趋势 | 12% | 这个品在涨还是在跌？ | 拼多多/淘宝 |
| 5 | 卖家集中度 | 10% | 有没有大卖家垄断？ | 闲鱼 |
| 6 | 利润率 | 10% | 每单能赚多少？ | 三方计算 |
| 7 | 跨平台价差比 | 8% | 信息差有多大？ | 三方计算 |
| 8 | 链接新鲜度 | 8% | 竞争在加剧还是稳定？ | 闲鱼 |
| 9 | 源平台评价质量 | 5% | 商品靠谱吗？退货多不多？ | 拼多多/淘宝 |
| 10 | 客单价区间 | 5% | 在不在甜区里？ | 三方 |

各维度评分规则详见附录A。

#### 智能定价策略

| 模式 | 触发条件 | 定价公式 |
|------|---------|---------|
| 信息差定价 | 闲鱼同款 ≤5个 | 采购价 × 1.4~2.0 |
| 竞争定价 | 闲鱼同款 6~20个 | 闲鱼TOP5均价 × 0.95 |
| 薄利定价 | 闲鱼同款 >20个 | 采购价 + 固定利润 或放弃 |
| 价格下限 | 通用 | 采购价 + 运费 + 手续费0.6% + 5元 |

#### 利润计算

```
闲鱼毛利润 = 售价 - 采购价 - 运费
闲鱼净利润 = 毛利润 - 手续费(0.6%) - 退货预留(8%)
```

### 3.2 小红书选品引擎（服务内容电商）

小红书和闲鱼的选品逻辑完全不同。闲鱼选品看"信息差"，小红书选品看"内容势能"。

#### 核心思维差异

```
闲鱼选品 (商品→市场):
  这个东西在闲鱼卖得少吗？→ 信息差大就做
  评判核心: 有没有人在卖？价格有没有差？

小红书选品 (内容→商品):
  这个东西能不能拍出好看的内容？→ 能种草就做
  评判核心: 这个品能不能让人刷到就想买？
```

#### 选品双路径

```
路径A: 从商品出发评估内容势能（主动选品）
  已有目标商品(来自比价引擎/手动添加)
  → 在小红书搜索相关内容 → 分析内容数据 → 给出"适不适合做小红书"的判定

路径B: 从内容热点反向找商品（趋势捕捉）
  监控小红书热搜/飙升话题/爆文
  → 发现"什么东西正在火" → 反向去拼多多/淘宝找货源 → 判断能不能做
```

#### 路径A: 从商品评估内容势能 — 五维度评分

| # | 维度 | 权重 | 核心指标 | 数据来源 |
|---|------|------|---------|---------|
| 1 | 话题热度 | 25% | 话题浏览量、30天笔记增量、增长率 | 小红书 |
| 2 | 内容可拍性 | 20% | 源平台评论区实拍图质量、品类视觉特征 | 拼多多/淘宝评论区 + AI判定 |
| 3 | 种草转化力 | 20% | 同品类互动率、收藏率、评论区购买意向占比 | 小红书 |
| 4 | 竞品笔记密度 | 20% | 30天新笔记数、带货笔记占比、头部笔记集中度 | 小红书 |
| 5 | 利润空间 | 15% | 小红书同品售价 vs 采购成本 | 三方计算 |

**各维度评分规则详见附录B。**

#### 路径B: 从内容热点反向选品 — 热点发现引擎

```
数据源1: 小红书搜索热词
  监控"搜索发现"页、搜索下拉推荐词
  重点关注近3天新出现或飙升的关键词

数据源2: 飙升话题
  按品类监控话题浏览量日环比增长
  增长率 >30%/天 → 标记为"飙升"

数据源3: 爆文分析
  定时扫描目标品类前50~100篇笔记
  突然出现的高互动笔记 → 可能预示新趋势

数据源4: 评论区挖掘
  高互动笔记的评论区出现频率最高的:
  "这是什么""哪里买的""求链接""已入手"
  → 这些商品就是有真实购买需求的

处理流程:
  热点/商品关键词 → 自动匹配拼多多/淘宝货源
  → 有货源: 进入路径A打分 → 推送到选品面板
  → 无货源: 标记为"暂无供应"，持续监控
```

#### 小红书爆文分析引擎

```
数据采集 (Playwright):
  搜索目标品类关键词
  → 采集前 50~100 篇笔记数据
  → 标题、封面图、正文、点赞数、收藏数、评论数、发布时间

分析维度:
  ┌─ 爆文特征提取 ──────────────────────────────────────┐
  │                                                      │
  │  1. 标题关键词频率分析                                 │
  │     哪些词出现在高互动笔记的标题里？                   │
  │     → 提取高频种草词（如"必入""绝绝子""闭眼买"）      │
  │                                                      │
  │  2. 封面图风格分析（CLIP辅助）                         │
  │     高互动笔记的封面是什么风格？                       │
  │     → 纯产品图 / 使用场景图 / 对比图 / 教程截图       │
  │     → 有无文字标注？什么颜色/字体效果最好？            │
  │                                                      │
  │  3. 内容结构分析                                       │
  │     高互动笔记是怎么写的？                             │
  │     → 测评型 / 教程型 / 种草型 / 合集型               │
  │     → 字数分布、emoji密度、分段方式                    │
  │                                                      │
  │  4. 发布时间分析                                       │
  │     什么时间发布的笔记互动率最高？                     │
  │     → 按星期×时段统计，精确到小时                     │
  │                                                      │
  │  5. 互动率计算                                         │
  │     互动率 = (点赞+收藏+评论) / 预估曝光              │
  │     高于品类均值 → 说明该品类内容有潜力                │
  │                                                      │
  │  6. 评论区意图分析（新增）                             │
  │     统计评论关键词:                                    │
  │     购买意向: "怎么买""链接""多少钱"                  │
  │     使用反馈: "已入手""好用""推荐"                    │
  │     → 计算"购买意向评论占比"                          │
  │                                                      │
  └──────────────────────────────────────────────────────┘

输出:
  · 品类内容潜力评分
  · 推荐的笔记风格和标题模板
  · 最佳发布时间建议（精确到星期×时段）
  · 可参考的爆文案例（含链接）
  · 建议的封面图风格
```

#### 小红书选品与闲鱼比价引擎的协同

```
两个引擎共享同一个商品数据库，但各自产出不同的推荐:

              ┌── 商品数据库(拼多多/淘宝) ──┐
              │                              │
```

## 四、销售助手：闲鱼模块

### 4.1 图片素材流水线

三级图片来源，逐级降级：

```
优先级1: 源平台评论区买家实拍图
  Playwright 抓取拼多多/淘宝评论区
  筛选: 有图好评 + 图片清晰度高
  提取 20~30 张候选实拍图

优先级2: 店铺商品详情图
  主图(5~8张) + 详情页展示图

优先级3: 用户上传实拍图（手动入口）
  买了样品后自己拍的，优先级最高
  一旦上传则替换自动获取的图片
```

智能加工，每次自动生成10张候选图：

```
基础处理（每张都做）:
  · EXIF元数据清除 + 重新压缩（改变文件哈希）

差异化处理（随机组合）:
  · 裁剪比例: 3:4 / 1:1 / 4:3 随机
  · 亮度/对比度/饱和度微调 (±3~8%)
  · 轻微旋转 (±0.5~1.5°)
  · 背景模糊/虚化
  · 文字标注（卖点/型号/对比标签）
  · 拼图组合（多张原图拼一张对比图）

小红书专用风格化:
  · 封面图加标题文字（大字报风格）
  · 生成对比图（使用前 vs 使用后）

输出 → 用户从10张中挑选 → [重新生成] [上传实拍替换] [确认]
```

```
→ 人工挑选图片 + 预览确认（可微调）
→ 系统通过Playwright执行发布
→ 记录发布状态和链接
```

### 4.3 账号健康度管理

#### 账号生命周期

| 阶段 | 周期 | 每日发布量 | 重点 |
|------|------|-----------|------|
| 养号期 | 第1~2周 | 0个 | 浏览/收藏/聊天，建立正常画像 |

- 发布: 5~8个（硬上限）

- 异常自动暂停机制

### 4.4 多账号管理

#### 账号矩阵策略

每个账号聚焦垂直品类（人设号），不做杂货铺：
- 品类不重叠 → 降低关联风险
- 垂直人设 → 买家信任度高、搜索权重加成
- 容灾分散 → 封一个号只丢一个品类

#### 同一身份多号安全准则

- 品类必须不同
- 网络/设备完全隔离
- 一个号出问题时全部降低频率
- 企业号可承载更大发布量

## 六、账号与网络隔离方案

### 6.1 多平台账号矩阵

同一身份可以同时在闲鱼+小红书+抖音开店，三个平台互不通信：

| 身份 | 代理IP | 闲鱼 | 小红书 | 抖音 |
|------|--------|------|--------|------|
| 身份1(你) | IP-1 或家庭宽带 | 闲鱼A | 小红书A | 抖音A |
| 身份2 | IP-2 | 闲鱼B | 小红书B | 抖音B |
| 身份3 | IP-3 | 闲鱼C | 小红书C | — |
| 公司 | IP-4 | 闲鱼D(企业) | 小红书D(企业) | 抖音D(企业) |

跨平台同IP安全，同平台多号必须不同IP。

### 6.2 代理IP方案

采用购买独享住宅代理IP方案，便于远程管理和扩展：

| 用途 | 方案 | 参考价格 |
|------|------|---------|
| 账号操作(发布/交易) | 独享静态住宅IP，每身份1个 | ¥35~50/月/个 |

| 服务商推荐 | IPIPGO / 快代理 / 站大爷 | 先试用再付费 |

所有IP选广东省内城市（广州/深圳/东莞/佛山），与实际位置匹配。

### 6.3 Playwright 环境隔离

每个身份独立的浏览器上下文：独立代理IP、独立设备指纹、独立Cookie存储、独立行为参数。加载 stealth 插件抹除自动化特征，禁用 WebRTC 防止IP泄露。

## 七、订单履约模块

### 7.1 履约总流程

```

              │
              ▼

              │
              ├── 校验通过 + 免密支付范围内 → 自动下单支付

              │
              ▼
       物流追踪 → 签收确认 → 订单完成
```

### 7.2 订单检测

```
检测方式: Playwright 轮询各平台订单管理页面
检测频率: 每5分钟
检测目标: "待发货"状态的新订单

检测到新订单后:
  · 提取: 订单号、商品信息、买家昵称、收货地址、付款金额、SKU
  · 与系统已发布商品做匹配 → 找到对应源平台商品
  · 写入数据库，状态: "待采购"
  · 推送钉钉通知: "[闲鱼A] 新订单 XX商品 ¥128，买家地址: XX"
```

### 7.3 源平台自动下单

#### 支付方案: 支付宝小额免密支付（自动化程度最高）

开启路径: 支付宝 → 设置 → 支付设置 → 小额免密支付（建议额度 ¥200）

目标品类采购价大部分在 ¥30~200 区间，Playwright 操作到付款页面后直接点击"确认支付"，支付宝免密自动扣款。

#### 下单流程

```
新订单入队
    │
    ▼
自动预处理 (即时):
  1. 通过发布时建立的映射关系，定位源平台商品链接和SKU
  2. 解析买家收货地址（姓名/手机/省市区/详细地址）
  3. Playwright 打开源平台商品页 → 选择SKU → 填写收货地址
    │
    ▼
安全校验（自动下单前必须全部通过）:
  · 采购价 vs 数据库记录价格，偏差 >10% → 暂停
  · 收货地址格式完整（省+市+区+详细地址+手机号）
  · 商品SKU与买家购买的一致
  · 源平台显示有库存
  · 单日累计自动支付未超过上限（默认 ¥2,000/天，可配置）
    │
    ├── 全部通过 + 金额 ≤ 免密额度
    │   → 自动点击支付 → 推送: "已自动采购 ¥45"
    │   → 订单状态: "已采购，待发货"
    │
    └── 任一不通过 或 金额超出免密额度
        → 暂停，推送: "订单XX需要人工确认: [原因]"
        → 等待人工处理（目标30分钟内）
        → 人工确认后 → 系统继续执行 或 手动输密码支付
```

#### 时效目标

| 场景 | 从买家付款到源平台下单 |
|------|---------------------|
| 全自动（校验通过+免密） | ~10分钟 |
| 半自动（需人工确认） | ≤30分钟 |

### 7.6 退货/售后处理

#### 商品描述退货政策

闲鱼商品描述统一附加声明: "非质量问题不支持退货退款"

#### 退货流程（实物商品）

```
系统检测到买家申请退货（Playwright轮询订单状态变更）
    │
    ▼
系统自动在源平台（拼多多/淘宝）发起退货申请
    │
    ▼
等待源商家同意退货（轮询检测）
    │
    ├── 源商家同意 → 系统获取源商家退货地址
    │   │
    │   ▼
    │ 推送钉钉通知:
    │   "买家XX申请退货
    │    商品: XX
    │    源商家退货地址: XX省XX市XX路XX号，联系人XX，电话XXX
    │    请在闲鱼跟买家沟通，让买家直接寄到以上地址"
    │   │
    │   ▼
    │ 你在闲鱼聊天里把退货地址发给买家
    │ 买家寄出后告诉你快递单号
    │   │
    │   ▼
    │ 你在系统中录入买家退货快递单号
    │ → 系统自动填入源平台退货信息
    │   │
    │   ▼
    │ 源商家签收 → 源平台退款给你
    │ → 你在闲鱼确认退款给买家
    │ → 订单状态: "已退货退款"
    │
    └── 源商家拒绝退货
        → 推送告警 + 建议处理方案
        → 人工决策
```

买家直接退到源商家，你不做中转，省时省事。

#### 退货注意事项

- 退货快递费: 非质量问题由买家承担，质量问题与源商家协商
- 闲鱼平台介入: 如果买家申请平台介入，可能判退（即使你写了不退货），系统需要同步处理源平台退货
- 虚拟商品退货: 原则上不退（已发送链接），如买家坚持可视情况处理

### 7.7 异常处理汇总

| 异常场景 | 系统行为 | 人工操作 |
|---------|---------|---------|
| 源平台缺货/下架 | 推送告警，暂停下单 | 联系买家协商换货或退款 |
| 源平台涨价 >10% | 暂停下单，计算新利润 | 确认是否继续或退款 |
| 买家地址解析失败 | 推送原始地址 | 人工修正后重新触发下单 |
| SKU匹配异常 | 暂停下单 | 人工确认正确SKU |
| 源平台发货超时 >72h | 推送告警 | 催促源商家或准备退款 |
| 物流异常(退回/丢件) | 推送告警 | 联系源商家售后 + 处理买家 |
| 买家退货 | 自动获取源商家退货地址 | 沟通买家退货到源商家 |
| 买家投诉/差评 | 高优先级推送到客服中心 | 优先处理 |
| 单日支付超上限 | 暂停自动支付 | 人工逐单确认 |

### 7.8 资金流与周转

```
资金流向:
  买家付款 ¥128 → 平台代管（闲鱼/小红书暂扣）
  你支付采购 ¥45 → 源平台商家（你先垫付）

  小红书: 确认收货后 T+7 天

周转资金需求:
  日均15单 × 采购均价¥50 × 放款周期10天 = ¥7,500
  建议预留: ¥10,000 流动资金
```

## 附录B：小红书五维度评分详细规则

### 1. 话题热度 (25分)

| 指标 | 得分(/25) |
|------|----------|
| 相关话题浏览量 >1亿 且 30天笔记增长 >15% | 25 |
| 浏览量 >5000万 且 增长 5~15% | 18 |
| 浏览量 >1000万 且 增长 >0% | 12 |
| 浏览量 <1000万 或 负增长 | 5 |

### 2. 内容可拍性 (20分)

| 指标 | 得分(/20) |
|------|----------|
| 产品上镜 + 多种内容形式(教程/测评/场景) + 评论区实拍丰富 | 20 |
| 产品上镜 + 至少2种内容形式 | 14 |
| 产品一般 + 需要场景布置才好看 | 8 |
| 产品不上镜 / 难以拍出差异化内容 | 3 |

### 3. 种草转化力 (20分)

| 指标 | 得分(/20) |
|------|----------|
| 品类互动率 >4% 且 购买意向评论 >30% | 20 |
| 互动率 2.5~4% 且 购买意向评论 15~30% | 14 |
| 互动率 1~2.5% | 8 |
| 互动率 <1% | 3 |

互动率 = (点赞+收藏+评论) / 预估曝光
购买意向关键词: "怎么买""链接""多少钱""已入手""求推荐"

### 4. 竞品笔记密度 (20分)

| 指标 | 得分(/20) |
|------|----------|
| 30天新笔记 <50篇 且 带货笔记 <10篇 | 20 |
| 新笔记 50~200篇 且 带货笔记 <50篇 | 14 |
| 新笔记 200~500篇 | 8 |
| 新笔记 >500篇 (红海) | 3 |

### 5. 利润空间 (15分)

| 指标 | 得分(/15) |
|------|----------|
| 预估利润率 >60%（内容溢价高） | 15 |
| 利润率 40~60% | 11 |
| 利润率 20~40% | 7 |
| 利润率 <20% | 3 |

小红书利润率 = (小红书可定价 - 采购价 - 运费 - 平台佣金) / 采购价

### 决策阈值

| 评分 | 决策 |
|------|------|
| ≥80 | 强烈推荐，优先出内容 |
| 60~79 | 值得做，安排内容排期 |
| 40~59 | 观望，话题起来再做 |
| <40 | 不适合小红书 |
